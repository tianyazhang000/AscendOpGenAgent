# Int32 向量加法

## 概述

在 NPU 上进行向量加法操作时，使用 `int32` 数据类型会比 `int64` 类型性能更优。原因在于 VADD 指令不支持 `int64` 类型，会退化为 scalar 加法，导致性能显著下降。**通过将 int64 数据类型转换为 int32**，可以启用 NPU 的向量加速单元，获得显著的性能提升。

## 触发条件

**当代码中存在以下场景时，应考虑应用此优化**：

1. **向量加法操作**：涉及两个向量的加法运算（如 `a + b`）
2. **数据类型为 int64**：`tl.int64` 或 Python 的 `int`（在 Triton 中默认为 int64）
3. **数据范围确定在 int32 范围内**：必须确保转换不会导致数据溢出

**重要约束**：只有当开发者确认数据范围在 int32 之内（`[-2^31, 2^31-1]`）时，才能启用此优化方法。

## 优化方法

### 原始代码（scalar 计算）

```python
a = tl.load(ptr_a + offsets)  # 类型为 int64
b = tl.load(ptr_b + offsets)  # 类型为 int64
c = a + b  # 退化为 scalar 计算
```

### 优化后代码（vector 计算）

```python
a = tl.load(ptr_a + offsets).to(tl.int32)  # 转换为 int32
b = tl.load(ptr_b + offsets).to(tl.int32)  # 转换为 int32
c = a + b  # 启用 vector 计算 (VADD 指令)
```

### 仅在 load 时转换（推荐）

```python
a = tl.load(ptr_a + offsets).to(tl.int32)
b = tl.load(ptr_b + offsets).to(tl.int32)
result = a + b  # vector 计算
# 如果需要返回 int64，再转换回去
c = result.to(tl.int64)
```

## 关键点

1. **数据范围验证**：必须确认原始数据在 `[-2147483648, 2147483647]` 范围内，否则转换会导致数据溢出
2. **转换位置**：在参与 VADD 运算前进行转换即可，无需全程保持 int32
3. **精度影响**：转换前后数值不变（当数据在 int32 范围内时），不影响计算精度
4. **适用场景**：向量加法、向量减法等逐元素运算

## 性能收益

将 int64 运算转换为 int32 运算后：
- 启用 VADD 指令的向量加速
- 避免退化为 scalar 计算
- 可获得数倍至数十倍的性能提升（取决于数据规模和运算复杂度）

## 风险提示

| 风险 | 说明 | 缓解措施 |
|------|------|----------|
| 数据溢出 | 超过 int32 范围的数据会溢出 | 仅在确认数据范围内时使用 |
| 精度问题 | 浮点数场景下可能引入精度变化 | 确保数据范围在安全区间 |
